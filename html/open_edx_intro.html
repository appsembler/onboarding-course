The Open edX project is a web-based platform for creating, delivering, and analyzing online courses. Read the Open edX Architecture guide to get to know the Open EdX architecture a little better before going forward on this chapter. These slides and speaker notes are a useful big picture overview of the components and scope of open source edX platform from which OpenCraft's client focused work is based.

Open edX is comprised of many different repos, with the most important ones being:

  • edx-platform: the core software that powers Open edX. This includes the LMS (student-facing, delivering courseware), and Studio (course authoring) components.
  • configuration: a collection of Ansible playbooks, scripts and utilities used to deploy Open edX.
  • devstack: a docker-based stack for developing or extending Open edX services.

Regarding Python versions, edx-platform (e.g. LMS or Studio) and our own Ocim product (OpenCraft Instance Manager) are Python 3, as are some newer Open edX components which run as independently deployed applications (e.g. blockstore). Many edx-platform dependencies (e.g. xblock, completion) are bi-compatible with Python 2.7 (using six) and get tested on both versions. Details are in the tox.ini for those repos. The forum service is coded in Ruby.

Open edX periodically releases stable versions of the platform called named releases. The OpenCraft maintenance contracts include upgrades to these new major releases. Since not every instance/client uses the same release, it's useful to know how the releases work and what's the current one. Take a moment to skim Open edX Named Releases before setting up your development environment. The edx.org site uses continuous deployment and we are considering moving to continuous deployment ourselves.
