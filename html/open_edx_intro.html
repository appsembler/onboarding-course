The Open edX project is a web-based platform for creating, delivering, and analyzing online courses. You can get to know the Open EdX architecture a little better here, take a look before going forward on this chapter. These slides and speaker notes are a useful big picture overview of the components and scope of open source edX platform from which OpenCraft's client focused work is based.

We recommend taking each repository as it comes into play with your tasks. To begin, these are the repos cloned by the devstack.

The main one is edx-platform. It is huge and has recently focused on putting dependencies into other repos. It relies directly on cs_comments_service, the Ruby language Discussion forum, and xqueue, a frontend to the 3rd party task queue software.

Other peripheral utilities are course-discovery and ecommerce, which are more interrelated than they should be, but not required for the base platform. The rest are utilities and plugins. The src directory (in the devstack) is to store local versions of XBlocks or plugins when you are working on them.

A final major repo is edx/configuration, which holds Ansible code to provision new servers with a running edx installation.

Regarding Python versions, edx-platform (e.g. LMS or Studio) is python2.7. Our own Ocim product (OpenCraft Instance Manager) is python3, as are some newer Open edX components which run as independently deployed applications (e.g. blockstore). Many edx-platform dependencies (e.g. xblock, completion) are now bi-compatible, using six, and get tested on both versions. Details are in the tox.ini for those repos. The forum service is coded in Ruby.

OpenEdX periodically releases stable versions of the platform called named releases. Since not every instance/client uses the same release, it's useful to know how the releases work and what's the current one. Take a moment to read Open EdX Named Releases before setting up your development environment. The edx.org site uses continuous deployment. OpenCraft maintenance contracts include upgrades to new major releases of Open edX. We are considering moving to continuous deployment ourselves.
