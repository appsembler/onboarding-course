A developer stack (or devstack) is a complete working install of Open edX on your computer. With it, you make changes to code locally and view those changes by visiting urls in your browser.

There are many versions of the edX devstack, but we can separate them into two categories:

  • Docker devstacks
  • Vagrant devstacks

While at some point in time you are likely going to need both types working, you should focus on getting the Docker devstack with the current release of Open Edx up first.

Much of the work that Open-Craft creates ends up being upstreamed to edx/edx-platform. This repository is the latest code and works very well with the newer Docker devstack. While the Vagrant devstack works as well for master work, it can be complicated and many people have had issues with getting it to work.

For many client projects, the Vagrant devstack is a must. McKinsey Apros and legacy systems that are not using the latest edx/edx-platform require the use of a Vagrant devstack. Until you are working on one of these tickets, or have some free cycles to put towards downloading and waiting it may be best to avoid setting up the Vagrant devstack.

Having said that, until things change, you'll eventually have many devstacks set up on your local machine. But for now, focus on the ones needed for your tasks, and if you're not sure which one to choose, please ask the reviewer of your on-boarding JIRA task.

The following sections have instructions on how to set up each devstack, feel free to skip the ones you won't use for now and come back to them when you need.

When provisioning devstacks, keep in mind that they are not compatible with Windows systems, only Linux and Mac systems.
