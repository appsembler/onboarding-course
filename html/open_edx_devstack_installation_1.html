There are two versions of the edX devstack.

A Docker version and a deprecated Vagrant version. While at some point in time you are likely going to need both working, you should focus on getting the Docker devstack up first.

Much of the work that Open-Craft creates ends up being upstreamed to edx/edx-platform. This repository is the latest code and works very well with the newer Docker devstack. While the Vagrant devstack works as well for master work, it can be complicated and many people have had issues with getting it to work.

For many of the client branches the Vagrant devstack is a must. McKinsey Apros and legacy systems that are not using the latest edx/edx-platform require the use of a Vagrant devstack. Until you are working on one of these tickets, or have some free cycles to put towards downloading and waiting it may be best to avoid setting up the Vagrant devstack.

Having said that, until things change, you'll eventually need both working. If you're not sure which one to choose, please ask the reviewer of your on-boarding JIRA task.
