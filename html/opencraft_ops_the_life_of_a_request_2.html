Getting an Open edX Instance

Before one can make an HTTP request to anything, they need a valid instance. Customers usually get this by going to http://opencraft.com/hosting/ and picking an option. Let's assume they chose the beta test version.

By default, when registering for an Open edX instance through the Ocim registration process, you choose a subdomain under opencraft.hosting, e.g. my-openedx-instance.opencraft.hosting, provided that it isn't already taken by somebody else.

Customers can also configure their own domain, such as "subdomain.example.com", but they'll need to set specific DNS records for this, which requires a bit of collaboration with us to get it right.
