When working with any McKinsey tickets, it's important to know that they have their own heavily-customized fork of edx-platform based on ginkgo, and that's called the Solutions Fork. To be able to run and work on the solutions fork, you will need to set up a Solutions devstack. Fortunately, since there's a lot of developers working on this all the time, it doesn't have nearly as many issues to set up as a normal Vagrant devstack.

Aside from all the common components you find on a normal devstack, Mckinsey uses a custom frontend called Apros instead of LMS, and it needs to be provisioned after the solutions devstack it set up.

Read How to work with Mckinsey before continuing.

To set up a complete Solutions Devstack, follow the procedures below.

1. Follow the procedures on  Setting up the solutions devstack.

2. After that, you should have a working solutions fork devstack running on vagrant. Check if the LMS and Studio are working correctly.

3. The next step is to set up Apros, and you can do that by following Setting up McKinsey Academy (Apros) with a local LMS Installation.

Again, if you find any bumps while provisioning step 2, troubleshoot, fix the issue and run vagrant provision again. Some issues that you've faced while provisioning a normal vagrant devstack might happen here too.

If the LMS, Studio and Forum are correctly configured, you shouldn't have any issues setting up Apros.
