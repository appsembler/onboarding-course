Once you have your first technical task assigned by your mentor you can start working on it.

The simplest workflow would be:

  1. Change the JIRA status of the ticket you're going to work on to In Progress.
  2. Set the remaining estimate to be the number of hours you expect to work on it (which should be less than or equal to the Original Estimate; ping the reporter of the ticket and the epic owner if the Original Estimate isn't set).
  3. Do git checkout of the master branch in the repository you're going to work with. Pull from remote.
  4. Create a new branch of the master branch. You can name it like: your_name/name_of_the_new_feature.
  5. Implement a new feature.
  6. Test your code, both manually and with tests, and make sure your code passes quality checks and is generally high-standard and readable for a reviewer. See the unit about Coding standards.
  7. Push your branch to remote.
  8. Open a pull request on GitHub/GitLab.
  9. Change the JIRA status of your ticket to Internal review and ping your reviewer.

Important notes:

The scenario above describes the simplest possible workflow but please do note that workflows can differ drastically for different tasks and one needs to ultimately figure out what works best through experience.

Please read the documentation on task workflows and statuses before starting your work,to understand how workflows may differ for more specific scenarios; but again, you should be adaptable.

Please read Pull request process docs to understand how pull requests are posted, reviewed, and merged.
