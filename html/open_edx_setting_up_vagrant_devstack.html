The vagrant devstack is required to work on most of ginkgo instances of Open EdX, and setting it up may be a difficult task at first.

You get some background start by reading up Running Vagrant-based Devstack. To set it up, follow these steps:

1. Create a new folder to hold your vagrant devstack (e.g ~/vagrant_devtack). 

2. Set up the release environment variable (usually open-release/ginkgo.1)

export OPENEDX_RELEASE=open-release/ginkgo.1

3. Clone the Vagrantfile and install required virtual hosts plugin by running:

curl -OL https://raw.github.com/edx/configuration/$OPENEDX_RELEASE/vagrant/release/devstack/Vagrantfile

vagrant plugin install vagrant-vbguest

4. Provision the devstack with:

vagrant up

In this last step is where you'll find most issues while setting up the devstack, but many developers already went through the trouble of fixing most of them so you don't have to. There's lots of tips, tricks and fixes on how to solve Vagrant devstack issues on the links below:

  • Running Vagrant-based Devstack
  • Troubleshooting Vagrant
  • Tips & Tricks for OpenCraft developers
  • Devstack tips and tricks. General discussion

Whenever you face an issue, troubleshoot it, fix the cause and run vagrant provision again to restart the provisioning of the devstack.

If you are stuck, don't hesitate to ask on Mattermost, and/or ask on the #general channel on edX's Slack.
